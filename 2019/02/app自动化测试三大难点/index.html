App自动化测试三大难点

二月 26, 2019 发布在 自动化测试, UI自动化测试

App自动化测试稳定性差、可维护性差、执行效率低三大难点应对策略

稳定性差

UI测试的不稳定性主要来自两个方面，一个是被测对象的不稳定，一个是测试代码的不稳定

稳定性非常重要，不稳定的自动化测试，会让开发和团队成员失去信任，直接决定成败

被测对象的不稳定

在UI自动化测试中，由于盛行敏捷管理，需求变化快，导致UI经常变化

建议：准确把握介入时机，不要在项目早期介入UI自动化测试，等待版本相对比较稳定成熟后在开展UI自动化测试

测试代码的不稳定

测试代码的不稳定主要体现在经常定位不到元素，从而容易失败。有时候会有莫名其妙的原因导致测试失败

建议：

  • 元素查找方式使用更稳定的id，xpath等方式
  • 元素查找时间适当延长
  • 使用适用性更广、更通用、更稳定的断言方式，尽量不使用UI元素作为断言，推荐如Activity
  • 尽量使用逻辑验证，减少数据验证，数据验证更适合接口测试
  • 更新到最新的版本Appium，更稳定
  • 尽量不要用sleep
  • 保证稳定测试环境：网络、测试系统和被测系统、测试账号等

可维护性差

可维护性是UI自动化测试的另一个难点，直接决定效率。为什么录制的脚本都不算自动化测试，就是这个原因

建议：

  • PO设计模式
  • 抽取页面公共方法，抽取测试用例公共方法，抽取常用手势操作方法
  • 测试数据和测试用例分离，测试用例和业务逻辑分离
  • 同一个类中测试用例链式依赖，方便增减测试用例

执行效率低

UI执行效率慢，这个倒是客观原因，因为需要模拟用户真实操作

建议：

  • 科学的设计测试用例，用例不是越多越好
  • 升级Appium到新版本，执行时间减少三分之一
  • Appium执行器使用uiautomator2

App专项测试（一）——性能指标

二月 26, 2019 发布在 专项测试

App专项测试的常用指标和采集方法

什么是专项测试

专项测试，就是对软件的某个质量维度进行深入测试。比如性能测试、安全测试等都属于专项测试。本文主要介绍Android的性能测试

Android的性能测试项主要包括：

  • 资源消耗
  • 内存泄漏
  • 电量功耗
  • 启动、运行耗时
  • 网络流量消耗
  • 帧率
  • 过度绘制

测试工具：

  • 通过ADB调试命令获取Android提供的API接口数据
  • Android手机开发者模式中自带的工具
  • Android Studio 3.0 自带的Android Profiler
  • 第三方工具，如腾讯GT、网易Emmagee、科大讯飞iTest等

测试方法：

  1. 设计场景：不同的性能指标，测试场景是不一样的。尽量通过自动化工具或脚本操作，不方便自动化的场景通过手工操作
  2. 获取数据：获取CPU、内存、电量、网络流量、响应时间等，最好从多种不同设备上测试，取平均值
  3. 通过数据分析是否有异常，分析异常原因并解决问题

内存

获取内存数据

  1. 通过App包名获取App进程pid

    ps |grep packageName

  2. 获取内存数据

    dumpsys meminfo <pid>

CPU

获取CPU数据

  • 通过App包名获取

    dumpsys cpuinfo |grep <packageName>

  • 通过top命令查看cpu占用最高的前10个程序（-t 显示进程名称，-s 按指定行排序，-n 在退出前刷新几次，-d 刷新间隔，-m 显示最大数量）

    top -m 10 -s cpu

网络流量

通过pid获取流量数据

cat /proc/<pid>/net/dev

获取的数据类似表格的形式输出，有三个部分，左边Interface是端口，中间Receive是接收的流量，右边Transmit是发送的流量

GPU

过度绘制：界面的Activity重叠多层，可以打开开发者选项中的“显示GPU过度绘制”

验收标准：

  • 不允许出现黑色像素
  • 不允许出现4x过度绘制
  • 不允许存在面积超过屏幕1/4区域的3x过度绘制（淡红色区域）

FPS:以60帧/秒作为标准，每帧的时间消耗为：1帧/60秒 = 0.01667帧/秒，约等于16.7毫秒。这样才能保证流畅度

在Android中，每隔16ms就会判断是否有数据需要显示，如果一帧图像超过16ms还没绘制好，就等待下一个时间间隔，画面就不会刷新，人眼会感觉到动画不连续，造成卡顿的感觉

测试方法

  • 在手机开发者模式中，选择“GPU呈现模式分析”，选择“在屏幕上显示为条形图”，屏幕上就会出现条形图，显示的是绘制每一帧所花的时间，绿线是16毫秒线，只要不超过绿线，就满足了60帧每秒的标准

  • 开发者模式中，将“GPU呈现模式分析”设置为“在adb shell dumpsys gfxinfo中”。通过adb命令获取帧率信息并导出到本地PC

    adb shell dumpsys gfxinfo com.packageName >D:/fps.txt

    找到“Profile data in ms”部分数据，这里显示了不同activity的帧绘制时间，可以复制到Excel中，做成图表，方便分析

屏幕帧率：滑动操作屏幕显示的帧率

弱网测试

模拟不稳定的网络，如低速、延迟、丢包等，测试App在恶劣网络下的表现

测试方法：

  • 使用真实的SIM卡、运营商网络来进行测试（有的问题必须在真实的网络中才能发现）
  • 通过代理上网，从而控制网络，Facebook开源工具ATC

需要做的：

  1. 在应用中统一弱网环境下的界面样式、动画效果、提示语、交互逻辑等
  2. 给用户明确的提示，如网络错误、服务器错误、超时等
